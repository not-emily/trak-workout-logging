## Progress Today
- **Phase 1 (Foundation)** — Rails 8 API scaffolded with UUID PKs via core `gen_random_uuid()`, JWT auth with 30-day expiry and silent refresh, custom policy pattern (`PolicyResult` + `Authenticatable` + `Syncable` concerns + `ApplicationController` registry). Vite/React 19/TS/Tailwind 4 frontend with `useAuth` hook, `apiClient` (snake↔camel + `X-Refreshed-Token`), pill BottomNav with framer-motion `layoutId` morph
- **Deployed to production** — Cloudflare Pages (`trak.1bit2bit.dev`) + Cloudflare Tunnel through farley_station (`trak-api.1bit2bit.dev` → `localhost:3002`) via launchd plist. Found and documented the two-cloudflared-configs gotcha (`/etc/cloudflared/config.yml` is what the daemon reads, not `~/.cloudflared/config.yml`)
- **Phase 2 (Exercise Library)** — Curated 79 exercises from `yuhonas/free-exercise-db` via reproducible `scripts/curate_exercises.rb`. Idempotent seeds. ExercisePolicy with system/owned visibility; system exercises read-only with `:system_exercise_readonly` reason. Frontend browser with search + kind + muscle-group filters, custom exercise add/edit
- **Phase 3 (Sync Layer + Session Logging — the MVP)** — Local-first sync layer (`localStore` with stable array refs cached + invalidated on mutation, `queue` append-only FIFO, `syncWorker` one-at-a-time drain with [1s/5s/30s/2m/10m] backoff, `schema` cache invalidation). Session/SessionExercise/WorkoutSet models (renamed from `Set` to dodge Ruby stdlib clash), policies, controllers, serializers, 23 tests. Active session UI with editable name, live elapsed-time, SessionExerciseBlock + SetRow (kind-aware fields), RestTimerBar (auto-starts on set check), AddExerciseSheet picker. Session history with summary cards. RetroactiveSessionPage. ActionMenuSheet wired to "+" satellite
- **Phase 4 (Routines)** — Routine + RoutineExercise models with cascade destroy, policies, controllers, 17 tests. Frontend types + actions (including `startSessionFromRoutine` that materializes session + nested + pre-filled sets without touching the template). RoutineDetailPage with editable name/description, drag-sortable exercises via @dnd-kit, kind-aware PlannedSetsEditor. ActionMenu now lists user's routines above empty/log-past
- **Bug fix** — `useAuth`'s `getSnapshot` was returning a new object per call, triggering React 19's "infinite loop" detection in production (silent — error swallowed in prod). Cached snapshot at module level, invalidate on `emitChange()`. This is the canonical `useSyncExternalStore` pattern; applied the same idea to localStore's `list()` to keep array references stable
