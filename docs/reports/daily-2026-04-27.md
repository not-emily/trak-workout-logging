### Deploy-pipeline fixes (commits `bb1737e`, `cc30265`)
- Cloudflare Pages started failing with "Missing: @emnapi/core@1.10.0 from lock file" after pushing the Phase 5 batch. Root cause: CF Pages bundles `npm@10.9.2` regardless of Node version, while my local lockfile was generated by `npm@11` (paired with Node 25). npm 11 changed how lockfiles handle conditional optionalDeps — leaves transitively-optional packages out of the `packages` map. npm 10's strict `npm ci` doesn't tolerate that
- Trak surfaces this specifically because Vite 8 uses Rolldown, which ships native bindings via `@napi-rs/wasm-runtime` → `@emnapi/{core,runtime}`. Garnish doesn't hit this because it's still on Vite ≤6 / Rollup (pure JS, no native bindings, no @emnapi)
- Fix #1 (`bb1737e`): added `frontend/.nvmrc` pinning Node 24. Got CF Pages onto Node 24 successfully but didn't change npm — CF bundles npm separately
- Fix #2 (`cc30265`): regenerated `package-lock.json` with `npx -y npm@10.9.2 install`. The new lockfile has the `node_modules/@emnapi/{core,runtime}` entries CF's npm 10 expects. `tsc -b` and `vite build` both clean locally
- Long-term: setting `NPM_VERSION=11` in CF Pages env vars (user did this) means trak will permanently use npm 11 going forward, matching local — prevents drift on future installs
